## 📖 О проекте

**Nano Banana Bot** — это не просто очередной AI-ассистент. Это бот с полным отсутствием морали, фильтров и ограничений (в рамках системного промпта). Он создан, чтобы отвечать максимально грубо, агрессивно и использовать жесткий мат, при этом предоставляя доступ к самым современным нейросетям для генерации контента. 

Бот умеет генерировать изображения (давая выбор между моделями Google и OpenAI), анализировать видео покадрово и вести контекстные диалоги, запоминая историю общения.

## ✨ Главные фичи

- 🤬 **Токсичная личность**: Системный промпт настроен на максимальную агрессию. Бот не извиняется, не церемонится и отвечает коротко и злобно.
- 🎨 **Генерация изображений (/image)**:
  - Выбор модели "на лету" через In-line кнопки: **Gemini 3.1 Flash Image** или **GPT-Image-2** (OpenAI).
  - Поддержка генерации на основе исходного изображения (Image-to-Image).
  - Автоматическое объяснение ошибок генерации (если запрос заблокирован фильтрами NSFW/Копирайта).
- 🧠 **Продвинутое "мышление" (Dynamic Thinking)**: Gemini использует режим глубокого размышления (`thinkingBudget: -1`) перед тем, как выдать ответ.
- 🎞️ **Анализ видео и GIF**: Бот разбивает отправленные видео на кадры (с помощью `ffmpeg`) и скармливает их нейросети вместе с аудиодорожкой для полного анализа происходящего.
- 💾 **Долговременная память (SQLite)**: Бот запоминает последние 10 сообщений диалога для каждого пользователя/чата. При перезапуске бота контекст не теряется.
- 🛡️ **Защита от спама (Cooldowns)**: Встроенная система лимитов (1 фото раз в 15 сек, 1 текстовый запрос раз в 5 сек).
- 💻 **Шедевральный код по запросу**: Несмотря на свой скверный характер, если пользователя прямо попросить написать код, бот выдаст профессиональный, рабочий и красивый скрипт без заглушек.

## 🏗️ Архитектура проекта

Проект полностью рефакторизован и разбит на логические модули для удобства поддержки и масштабирования:

```text
NanoHatani/
├── main.py            # Точка входа, запуск бота и роутера
├── config.py          # Токены, ID чатов, системные промпты
├── state.py           # In-memory хранилище (кулдауны, стейты кнопок)
├── database.py        # Инициализация и работа с SQLite (bot_data.db)
├── keys_manager.py    # Продвинутый парсинг и ротация API ключей из r.txt
├── utils.py           # Вспомогательные функции (проверка подписки на чат)
├── ai_services.py     # Вся логика запросов к API Gemini и OpenAI (aiohttp)
├── handlers.py        # Роуты (команды, обработка текста, видео и коллбеков)
├── r.txt              # Файл с API ключами (добавляется в .gitignore)
└── bot_data.db        # База данных истории чатов (создается автоматически)
```

## 🚀 Установка и запуск

### Требования
- **Python 3.10+**
- Установленный **FFmpeg** (для анализа видео: `sudo apt install ffmpeg`)
- **PM2** (опционально, но рекомендуется для фонового запуска)

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/Rewixx-png/NanoBananaBot.git
cd NanoBananaBot
```

### Шаг 2: Установка зависимостей
```bash
pip install aiogram aiohttp aiosqlite
```

### Шаг 3: Настройка API ключей
Создайте файл `r.txt` в корне проекта. Бот умеет парсить как чистый текст, так и JSON блоки.
Пример содержимого `r.txt`:
```json
{
  "gemini": [
    "AIzaSyВашПервыйКлючGemini...",
    "AIzaSyВашВторойКлючGemini..."
  ],
  "openai": "sk-proj-ВашКлючOpenAI..."
}
```
*(Если один из ключей Gemini исчерпает лимит или будет забанен, бот автоматически удалит его из файла и перейдет к следующему).*

### Шаг 4: Конфигурация
Откройте файл `config.py` и вставьте свои данные:
- `BOT_TOKEN` — токен вашего Telegram-бота (от @BotFather).
- `CHAT_ID` — ID обязательного чата/канала для подписки (чтобы бот работал, пользователь должен в нем состоять).

### Шаг 5: Запуск
Для обычного запуска в консоли:
```bash
python3 main.py
```

Для работы 24/7 (через PM2):
```bash
pm2 start main.py --name NanoBananaBot --interpreter python3
pm2 save
```

## 🛠️ Доступные команды

- `/start` — Приветственное сообщение и проверка доступа.
- `/image [промпт]` — Генерация изображения. Вызывает In-line клавиатуру для выбора движка (Gemini / GPT).
- `/clear` — Принудительная очистка истории диалога (БД) для текущего чата.
